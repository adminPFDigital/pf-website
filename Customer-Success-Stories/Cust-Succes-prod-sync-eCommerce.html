Product Sync for eCommerce


About The Project :


Client is a Fortune 500 company that sells Fashion products via e-commerce site. They use a 3rd Part Logistics system to fulfill the orders placed by the customer and wanted a solution to integrate their e-commerce site with the 3PL system. Also need a reporting application to monitor the data, correct and re-sync the data in case of any validation errors or connectivity failures


Scope Of The Project :


  • Process to sync Orders and Customer data from e-commerce sales channel to an intermediate database.
  • A scheduled process to send the data from Database to 3PL system via EDI(850) to trigger the Order Fulfillment
  • A scheduled process to receive Shipments data from 3PL system via EDI(856) and update it to e-commerce system
  • A scheduled process to receive Inventory feed from 3PL system vis EDI(846) and update it to e-commerce system
  • APIs to build reporting screens to view, edit and re-try sync for failed records along with email notifications

Challenges :


  1. Loss of EDI file due to connectivity issue
  2. Building a 3-step process to process the input file.
  3. Identify exact state from address as Customer may enter state code or name and is prone to typo errors.
  4. Separate API to validate the Zip Code using Google API to identify the exact state to accommodate the Custom logic needed in the integration
  5. Logic check to make sure there is no loss of events or files
  6. Scheduled process to read through all the webhook events and reconcile with e-commerce system and intermediate Database to find the missing records
  7. Scheduled process to read through the backup files to compare with the processed files to find the unprocessed file and re-process automatically based on the cause
